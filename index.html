Introduction

The HTML markup language is nearly 30 years old and over its lifetime it is grown significantly in complexity. However, at the same time web professionals have often skipped over learning basic HTML first in favor of immediately exploring design or programming frameworks. While such a choice would seem appropriate to get hands on experience, skipping markup basics can be limiting resulting in confusion or even . It is our hope that by approaching HTML using it is understandable that we would want to get things done.

Comments

HTML comments are the form of <!-- an HTML comment -->.

With HTML comments be careful with embedding the -, !, > and < characters.

HTML comments are useful for documenting features, showing authorship

HTML comments are seen by anyone who looks at the page source as such page authors are warned to avoid putting sensitive information in them or to strip the comments during a minification process.

HTML comments can be used to insert content that is inert and have it extracted using JavaScript. The use of data-* attributes, a template element or string literals or other data format within a script block is a preferred way to embed such data payloads.

Character Entities

A few example entities are < (less than &lt;), > (greater than &gt;, & (ampersand &amp;, © (copyright &copy;) and numeric entities like ☺ (a unicode smile). Many many entities exist the main point is to understand the syntax and look up the symbol needed.

The Document Element

The html element surrounds the complete document and is the root of the parse tree.

The two allowed elements within the html element are the head element where we keep information about the page and its content and the body element which contains the actual content itself.

The Document Metadata

The metatag will be found in the head which is often used to set the character set of the document which should be the first thing set. (See the head of the document) The meta tag also can be used to set keywords, simulate the equivalent of HTTP response headers and other configuration duties.

The head should contain a title tag which is the only mandatory HTML element for a document. The title should be set to a descriptive value as it is used in book marks and search.

The head may also contain a variety of link tags which may be used to asscoiated a style sheet or define various navigational relationships. The style of this element was set by an exeternal style sheet. We also can use a style element in the head itself. This element style was set by an internal style block. The global attribute style also set style information inline. The choice between linked, document wide, or inline style varies based upon interest in performance and maintainability. Generally you are encouraged to separate concerns as much as possible.

    Table Time!

    HTML tables used to be the backbone of design, today they are back to their normal use. Given that data powers our world we should (re)discover what features HTML tables provide and then and only then turn to libraries to help us present data in a tabular manner. Understand that if we want to interact with data we call that a datagrid which is a different thing all together. For now datagrids are not native in HTML, but libraries like Handsontable and ZingGrid can certainly address that.

    Form Fun

    Intearctive forms in HTML initially solely powered posting data to server-side programs. Today forms are used to power complex client side applications written in JavaScript. Much of the markup complexity is found with the input element which does many things. However, other form elements can be quite complex themselves. Before moving on to JavaScript make sure you fully understand all the details around form controls otherwise you will have gaps in your knowlege and may run into trouble or do needless scripting when you could just use platform provided features directly.
